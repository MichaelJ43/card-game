# Audio cues

The shell exposes three optional **sound cues** (top right, next to the **Card table** title). Each cue is stored in `localStorage` as an integer **volume 0‚Äì100**. The current UI uses a single toggle per cue: **off = 0**, **on = 100**, so the storage layer is ready for finer volume controls later.

| Cue | When it plays | Visibility |
|-----|----------------|------------|
| **Chat sound** | When a room chat line is shown as a **toast** on the main window (chat popout **closed**). | Toggle appears only while you are in an online room (host or joined client). |
| **Turn sound** | When the table‚Äôs **current player** changes (games that expose `gameState.currentPlayer`) **and** the document is not both visible **and** focused (another tab, another window, or the browser is in the background). | Always |
| **Card sound** | When the **table fingerprint** changes (any card moves, flips, or zone changes). Debounced slightly so rapid updates collapse to one cue. | Always |

Storage key: `card-game:audio-cues:v1` (JSON: `{ "chat": 0, "turn": 0, "flip": 0 }`).

Bundled assets live under `public/sounds/` as **WAV** files. They are short synthetic tones generated by `scripts/generate-tone-wavs.mjs` (`npm run gen:sounds`).
